# AgentCore 見積エージェント コスト方針

## 1. 基本方針

初期段階では、個人開発・PoC用途としてコストを抑えた構成を優先する。

常時課金が発生しやすいサービスや、大規模なRAG基盤は初期構成では利用しない。

## 2. 現在利用しているAWSリソース

| リソース | 用途 |
|---|---|
| Amazon Bedrock AgentCore Runtime | エージェント実行 |
| Amazon ECR | Runtime用コンテナイメージ保存 |
| AWS CodeBuild | AgentCoreデプロイ時のコンテナビルド |
| AWS IAM | 実行ロール・デプロイ権限 |
| Amazon CloudWatch Logs | Runtimeログ |

## 3. 現在の見積もり方式

現在のエージェントは、AWS公式料金APIではなく、コード内の簡易単価表を使って概算を返す。

この方式は以下を目的とする。

- 低コストで動作確認する
- 見積もりフローを早く作る
- UI/API連携の出力形式を固める
- 後でPricing APIやBedrock解析に差し替えやすくする

## 4. 現在の概算に含めていないもの

- 税
- サポート料金
- データ転送料金の詳細
- 無料枠
- Savings Plans / Reserved Instances
- リージョンごとの正確な単価差
- CloudWatch詳細メトリクスや大量ログ
- NAT GatewayやALBなど未対応サービスの料金

## 5. 今後の改善方針

1. よく使うサービスの簡易単価表を拡張する
2. インスタンスサイズやストレージ種別を入力から抽出する
3. AWS Pricing APIと連携する
4. 見積もり結果をS3に保存する
5. 本番/検証/開発環境別の合計を出す
